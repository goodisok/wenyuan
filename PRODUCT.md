## 1. 产品定义

### 1.1 一句话

**问元**是以「天地人三元」为框架的八字命盘 Web 服务：程序完成准确排盘，用户阅读结构化 AI 解读，并就**当前命盘**有限度地追问。

### 1.2 是什么 / 不是什么

| 是 | 不是 |
|----|------|
| 公网多用户命盘工具（PC + 手机） | 在线算命平台 / 社区 |
| 服务端统一提供 AI | 用户各自配置 API Key |
| 文化参考与自我觉察 | 医疗 / 投资 / 决策依据 |
| 锚定命盘的 L1/L2/L3 问答 | 脱离命盘的 AI 闲聊 |
| 无账号，URL + 本地历史 | 云端账号与命盘库 |

### 1.3 天地人三元

| 元 | 产品模块 |
|----|----------|
| **天元** | 起运、大运、流年、小运 |
| **地元** | 四柱、纳音、五行、旬空、刑冲合害 |
| **人元** | 十神、十二长生、胎元 / 命宫 / 身宫 |
| **问** | L1 六章解读 · L2 预设追问 · L3 就盘问答 |

### 1.4 设计原则

1. 程序算、AI 说  
2. 规则层摘要锚定 AI，禁止脱离命盘发挥  
3. 手动触发 AI，可全站关停  
4. 双端同能，布局可不同  
5. 生辰默认不落库；日志脱敏  
6. 温和表达，免责声明常驻

## 2. 用户、场景与规模

### 2.1 目标用户

- **P1** 命理爱好者：要准、要全、可分享  
- **P2** 好奇探索者：手机打开即用、能问具体问题  
- **P3** 开发者：REST API  

### 2.2 核心场景

1. 输入生辰 → 排盘 → 浏览三元命盘  
2. 触发 L1 解读（子平直断）→ L2 点选追问 → L3 自定义问答  
3. 复制链接分享；他人打开同一命盘（重新排盘，结果一致）  
4. 本机查看最近排盘历史（localStorage）  

### 2.3 规模目标（v1.0 设计指标）

| 指标 | 目标 |
|------|------|
| 同时在线 | ~50 |
| 日活（参考） | ~500 |
| 排盘 QPS | ~10（峰值） |

单机 2C4G 云服务器 + Nginx + Uvicorn 4 workers 可满足；超规模再扩容。并发能力主要受服务器与 DeepSeek 上游限制，**产品层不做访问限流**。

## 3. 部署与多用户

### 3.1 部署架构

```
用户浏览器 (HTTPS)
    ↓
Nginx / Caddy（TLS、静态缓存、反代）
    ↓
Uvicorn × 4 workers（FastAPI 无状态）
    ↓
DeepSeek API（服务端唯一 Key，.env 配置）
```

- 绑定 `0.0.0.0`，对外仅 443（与 80 跳转）  
- 配置项：`DEEPSEEK_API_KEY`、`AI_ENABLED`  
- 健康检查：`GET /health`（监控用）  
- **v1.0 不要求** Docker / CI；需提供《部署说明》文档  

### 3.2 多用户策略

| 层 | 策略 |
|----|------|
| 排盘 | 无状态，任意 worker 处理 |
| AI | 全站共享 Key；不设 IP 限流与并发上限 |
| 会话 | 无服务端 session；L3 历史由前端传递 |
| 存储 | 不落库；不存用户生辰与命盘 |

### 3.3 管理开关

`AI_ENABLED=false` 时 `analyze` / `ask` 返回 503 +「AI 解读暂时关闭」。用于维护或上游异常时手动关停。

**产品层不做 IP 限流、QPS 限流或 AI 并发上限。**

### 3.4 隐私与合规

- 页脚 + `/privacy` 简述：生辰用于排盘与 AI，**服务端不持久化**；AI 请求会发往 DeepSeek  
- 分享链接 **含编码后的生辰**，复制前弹窗提示「链接包含出生信息，请谨慎分享」  
- 日志：不记录完整 `birth_date`、chart JSON；AI 错误仅记录 status code

## 4. 「问 · AI」交互（已定）

### 4.1 三层

| 层级 | 触发 | 接口 | 说明 |
|------|------|------|------|
| **L1** | 按钮「开始解读」 | `POST /api/analyze` | 六章 Markdown，SSE 流式 |
| **L2** | 解读下方 chips | `POST /api/ask` | 单次问答 |
| **L3** | 输入框 | `POST /api/ask`（带 history） | 就盘追问，SSE |

### 4.2 L2 预设问题（已定）

- 展示 **4～6 个** chips  
- **2 个固定**：「当前大运对我意味着什么？」「我的五行平衡吗？」  
- **2～4 个动态**：由 BaziInsight 生成（如旺火、当前大运干支）  
- 点击即发问，结果插入问答区，不重新跑 L1  

### 4.3 L3 会话（已定）

- **1 轮** = 用户 1 条问 + AI 1 条答  
- 上限 **8 轮** / 命盘 / 标签页；第 9 条提示「本轮已达上限，请刷新或重新打开命盘」  
- 关闭标签页会话清空；可选写入 localStorage 仅本机只读恢复（v1.0 做）  
- 拒绝与盘无关问题：AI 模板内约束 + 后端检测空 chart 拒绝  

### 4.4 AI 上下文（每次请求顺序）

1. 系统角色与边界  
2. BaziInsight 规则摘要（JSON）  
3. 命盘 chart 精简摘要  
4. 已有 L1 解读全文（若有）  
5. L3：`history` 最近 8 轮  
6. 当前 `question`（L2/L3）  

### 4.5 明确不做

- 无 chart 的聊天  
- 独立全屏聊天页  
- 无限轮次  
- 用户自带 DeepSeek Key

## 5. 数据：URL 与 localStorage（已定）

### 5.1 URL 分享

**路径：** `/chart?s={payload}`  

**payload：** Base64URL(JSON)，仅含排盘输入，**不含** chart 结果：

```json
{
  "v": 1,
  "date_type": "solar|lunar",
  "birth_date": "1990-05-15",
  "birth_time": "12:30",
  "gender": "male",
  "is_leap_month": false
}
```

- 打开链接 → 前端解码 → 调 `POST /api/chart` → 渲染（与直接排盘一致）  
- 分享的是「同一张盘的入口」，不是静态 HTML 快照  
- 复制链接前显示隐私提示  

### 5.2 localStorage

| Key | 内容 | 限制 |
|-----|------|------|
| `wenyuan_history` | 最近排盘输入列表 `[{input, label, ts}]` | 最多 **20** 条，FIFO |
| `wenyuan_ai_{hash}` | 该命盘的 L1/L2/L3 缓存 | 每盘 1 条，hash=输入参数 |
| `wenyuan_chart` | 当前会话 chart（过渡，兼容） | 与 URL 恢复并存 |

`label` 示例：`1990-05-15 男 · 庚午`（排盘后更新）。

## 6. 排盘与规则层规格（已定）

### 6.1 历法

- **公历：** `type=date` + `type=time`  
- **农历：** 年（1900–2100）、月 1–12、日 1–30、**闰月 checkbox**、`is_leap_month` 传 API  
- 无效日期返回 422 / `success:false` 中文原因  

### 6.2 命盘字段（v1.0 必展示）

- 四柱：干支、十神、纳音、藏干、藏干十神、旬空  
- 命主：性别、生肖、日主、十二长生、虚岁、阳历/农历生辰  
- 胎元、命宫、身宫  
- 五行统计 + **五行条形图**  
- **刑冲合害**（见 6.4）  
- 起运：起运岁数、起运年份、交运说明（来自 `lunar-python` Yun）  
- 大运：干支、年份、年龄、流年列表（可展开）  
- 小运：起运前列表  

### 6.3 虚岁（已定）

**传统虚岁：** `当前农历年 − 出生农历年 + 1`（与现有逻辑一致，排盘结果中标注「虚岁」）。  
v1.0 不做法定周岁/实岁双显示。

### 6.4 刑冲合害（已定范围）

**展示四柱两两关系标签**（去重）：

- 地支：**六合、六冲、三刑**（含自刑）、**六害**  
- 天干：**五合**（甲己合等）  
- 不做：三合局、三会方、拱合等高级关系（v1.1+ 再议）  

### 6.5 BaziInsight 规则层

程序生成 `insight`，经 **`publish_*` 过滤**后由 `public_insight()` 返回前端；完整字段与模块映射见 [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) §4。

**前端可见（摘要）：** 观命总观、旺衰体用、调候、格局喜用、高置信断事、六亲人事印证、命局要点等。

**仅 AI 使用（不返回前端）：** `citations`、`corpus_meta`、完整 temperament 等。

**原则：**

- `day_master_strength` 等为倾向判断，非流派定论  
- 不做专业唯一用神；AI 不得脱离规则层断言

## 8. API 契约（已定）

### 8.1 `POST /api/chart`

**请求：**

```json
{
  "date_type": "solar",
  "birth_date": "1990-05-15",
  "birth_time": "12:30",
  "gender": "male",
  "is_leap_month": false
}
```

**响应：** `{ "success": true, "data": { chart..., "insight": {...} } }`  
chart 内含上述全部字段；`insight` 同 6.5。

### 8.2 `POST /api/analyze`（L1，SSE）

**请求：** `{ "chart": {...}, "insight": {...}, "style": "classic|modern" }`  

**响应：** `text/event-stream`  

```
event: chunk
data: {"text":"## 一、三元总览\n"}

event: done
data: {"analysis":"完整文本"}

event: error
data: {"error":"..."}
```

非 SSE 客户端可保留 JSON 一次性响应（`Accept: application/json`）作降级。

### 8.3 `POST /api/ask`（L2/L3）

**请求：**

```json
{
  "chart": {},
  "insight": {},
  "analysis": "已有L1全文或空",
  "question": "当前大运宜注意什么？",
  "history": [{"role":"user","content":"..."},{"role":"assistant","content":"..."}],
  "style": "classic|modern"
}
```

**响应：** 同 analyze，支持 SSE 或 JSON `{ "success", "answer" }`。

### 8.4 错误码

| HTTP | 含义 |
|------|------|
| 422 | 参数校验失败 |
| 503 | AI 关闭或上游不可用 |
| 200 + success:false | 业务错误（中文 message） |

## 11. 实施顺序

```
Phase 1  数据与分享
  URL 编解码 · /chart 恢复 · localStorage · 隐私提示 · /privacy

Phase 2  排盘与规则层
  闰月 · 旬空 · 起运 · 藏干十神 · 刑冲合害 · BaziInsight · chart 扩展

Phase 3  命盘 UI（响应式三元）
  分区布局 · 五行图 · 大运卡片/展开 · 移动 sticky 问区

Phase 4  问 · AI
  SSE · L1 接 insight · L2 chips · L3 /api/ask · 缓存 · 复制

Phase 5  公网与多用户
  AI_ENABLED · 部署文档 · 双端与压测验收
```

Phase 5 可与 Phase 4 部分并行；**Phase 2 阻塞 Phase 4**。
